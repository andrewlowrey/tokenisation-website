Assumptions
Technical Assumptions
  • Wallet remains the single source of truth for balance during gameplay. GIS does not call CRM Rewards directly during a session.
  • Supplier-facing API contracts remain unchanged. No supplier rebuild or redeployment required.
  • Mobile landscape EZ Nav component will serve as the desktop version for BetMGM tokenisation (no separate desktop EZ Nav maintained by CI team).
  • Subsequent game relaunches (after initial launch) will be faster due to geolocation caching — to be confirmed by tech teams.
  • RTMS events already published by In-Game Widget will be used to update the progress bar (per Santhosh Bethapudi).
  • API contracts from EDS/CRM will be finalised and stable before S11 dev begins.
  • 2 dedicated devs + 1–2 QA at 100% capacity enables 7–8 sprint delivery for GIS workstream.
Product & Commercial Assumptions
  • BetMGM has verbally accepted the full game reload on bonus switch. Written sign-off will be obtained before dev completes.
  • This project is BetMGM US only. No other region's EZ Nav is affected.
  • Casino development and integration testing runs Jan–end of July 2026. UAT in September/October. Production release Oct 2026 (Sprint 21).
  • CRM Studio offer creation is out of scope for GIS and GAMP teams — they only consume the APIs.
  • Free spins lock-in behaviour is consistent with current supplier game behaviour and does not require BetMGM sign-off.
  • All development and testing uses BetMGM-specific lower environments (QA6, QA8, QA9) only.
  • Phase 2.1 R2 (Universal Bonus / multi-product signposting) is a separate workstream and not in scope for this delivery.
